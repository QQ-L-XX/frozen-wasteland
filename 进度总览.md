## 已完成的 Codex 关键工作

### 稳定性与存档

- 修复 `unlocked` 蓝图解锁状态在保存/读取中丢失。
- 修复 QA 存读只做内存往返、不写本地存档。
- 修复 QA 存读覆盖本地存档时丢失 `unlocked`。
- 修复自动存档覆盖外层 UI 字段。
- 新增 `GameManager.saveLocal(extraSaveData)`，手动保存统一走安全写档。
- 修复旧档缺少 `greenhouse/world/outposts/production/longTerm/qa` 时直接损坏。
- 修复旧档缺少 `vehicle` 时载具变成 `undefined`。
- 修复旧档 `vehicleUpgrades/relations` 格式异常时直接损坏。
- 保存/读取教程步骤和背景故事触发记录，避免读档后重复弹出。
- 统一 3 个读档入口的 UI 同步，读档后刷新蓝图、工具状态、`window.game` 和已打开面板。
- 读档后补齐温室棋盘标记，避免旧档完成度面板不识别已建温室。
- QA 存读验证成功会把 `qa.saveLoadVerified=true` 写回本地存档，避免重开后完成度回退。
- 手动读档成功也会把验证状态写回本地存档，避免未等到自动存档就退出时状态回退。
- 读未结局存档时会清除当前局残留结局状态，避免完成度误判结局已完成。
- 读档前重置可选系统，避免旧档缺字段时保留当前局载具、温室、世界等脏状态。
- 修复搜刮空包撤离未清理 active 状态，第一周 P0 逻辑烟测通过。
- 一周 P0 烟测改为非破坏式：运行后恢复当前局和本地 `frost_save`，只返回烟测结果。
- 三个 UI 读档入口统一改为安全解析；坏档/截断档会提示“存档损坏，无法读取”，不再抛异常卡死。
- 新游戏、标题继续、系统读档、暂停读档、返回标题统一清理临时 UI，避免仓库/菜单/暂停层/搜刮层残留。
- 修正完成度/长线上下文的载具类型：无载具状态正式支持 `null`，避免后续完成度判断把“未造载具”当作异常。
- 修复读档时存档天气被 `weather.init(day)` 重算覆盖的问题，QA 存读验证不再对天气变化假通过。
- 2026-06-29 发布前复测：TypeScript 检查通过，P0 一周烟测通过且保持非破坏式。

## 百小时目标判断

现在不能说已经能稳定支撑 100 小时。

当前更接近：

- 认真玩：20-40 小时潜力。
- 补齐中期经营和世界网络：50-70 小时潜力。
- 补齐终局路线、长期灾变、角色线、挑战剧本：80-100 小时潜力。

要达到 100 小时，核心不是再堆随机事件，而是让玩家每 10-20 天都有新的长期目标、结构变化和风险升级。
