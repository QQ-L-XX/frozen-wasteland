# 幸存者 AI 行为树设计

> 关联 GDD 第七章 | 2026-05-18 | v0.1

---

## 一、设计目标

幸存者不是玩家的提线木偶。他们有自己的**自主决策**，但也接受玩家的**手动指令**。目标是环世界式的"你定方向，他们自己执行"的半自动化管理。

---

## 二、AI 架构：三层决策

```
【第三层】战略决策（每游戏小时评估一次）
  ├─ 我该做什么工作？
  ├─ 我的状态还行吗？（饥饿/疲劳/体温/士气）
  └─ 有没有紧急事件需要响应？

        ↓ 选定任务类型

【第二层】任务执行（持续）
  ├─ 建造任务：走到目标格 → 建造 → 寻找下一个同类任务
  ├─ 搜刮任务：出发到世界地图 → 进入场景 → 自主搜刮（如果是AI控制）
  ├─ 种植任务：走到温室 → 浇水/播种/收获
  └─ 搬运任务：从 A 点取物 → 送到 B 仓库

        ↓ 执行过程中持续检查

【第一层】即时反应（每帧/每秒）
  ├─ 温度跌破阈值？→ 找热源
  ├─ 敌人出现？→ 战斗或逃跑
  ├─ 受伤？→ 寻求治疗
  └─ 极度疲劳？→ 中断当前任务去睡觉
```

---

## 三、第三层：战略决策——工作选择

### 3.1 优先级表（玩家手动设定的核心）

```
       建造  狩猎  种植  医疗  维修  搜刮  搬运  研究  休息
老兵    3     4     /    2     2     1     3    /    -
技师    1     /     /    /     1     3     4    /    -
医生    /     /     2    1     /     /     3    /    -
农民    4     /     1    /     3     4     2    /    -
```

- `1` = 最优先，`/` = 永不做，`-` = 系统托管（按需求自动）
- 数字相同时 → 选择距离最近的可用任务

### 3.2 战略评估流程（每游戏小时）

```
开始战略评估
    │
    ├─ 检查紧急中断条件
    │   ├─ 体温 < 危险阈值？→ 寻找热源 [覆盖所有]
    │   ├─ 饥饿 > 80%？→ 进食 [覆盖所有]
    │   ├─ 疲劳 > 90%？→ 睡觉 [覆盖所有]
    │   └─ 濒死/重伤？→ 自我急救或去医务室 [覆盖所有]
    │
    ├─ 通过紧急检查 → 读取玩家设定的优先级表
    │
    ├─ 按优先级 1 → 4 → 6 → ... 顺序扫描
    │   └─ 对每个工作类型：
    │       ├─ 有可执行的任务吗？（存在未建造蓝图 / 成熟作物 / 待搬运物资...）
    │       ├─ 我的属性合格吗？（如医疗需要智力 ≥ 3）
    │       ├─ 任务在我可到达范围内吗？
    │       └─ 全部满足 → 执行该任务，结束评估
    │
    └─ 没有任何可用任务 → 进入"闲置"状态
        └─ 闲置行为：在基地附近随机走动、与其他幸存者社交、发呆
```

### 3.3 工作类型判定条件

| 工作 | 有可执行任务的条件 |
|------|--------------------|
| 建造 | 存在已规划但未完成的建筑蓝图或管道 |
| 狩猎 | 世界地图有可选区域 + 有武器 |
| 种植 | 温室有未浇水的作物 / 有空地可播种 / 有成熟作物 |
| 医疗 | 有受伤/生病的幸存者 + 医务室可用 |
| 维修 | 有损坏的建筑/管道/设备 |
| 搜刮 | 世界地图有未枯竭区域 + 有装备 |
| 搬运 | 地面有未入库的物资 / 仓库物资需要整理 |
| 研究 | 工坊有未解读的蓝图 / 书籍 |

## 四、第二层：任务执行——状态机

每个工作类型对应一个执行状态机。

### 4.1 建造任务状态机

```
        ┌─────────────┐
        │ 寻找最近的    │ ← 扫描所有未完成蓝图
        │ 建造蓝图      │
        └──────┬──────┘
               │ 找到
               ▼
        ┌─────────────┐
        │ 走到目标格    │ ← 寻路
        └──────┬──────┘
               │ 到达
               ▼
        ┌─────────────┐
        │ 检查材料      │
        │ 够吗？        │
        └──┬───────┬──┘
      不够 │       │ 够
           ▼       ▼
    ┌──────────┐ ┌──────────┐
    │ 去仓库    │ │ 开始建造  │ ← 进度条，持续时间 = 建造量/力量
    │ 取材料    │ └────┬─────┘
    └────┬─────┘      │ 进度条满
         │ 取到        ▼
         └──────→ ┌──────────┐
                  │ 建造完成  │
                  └────┬─────┘
                       │
                       ▼
                  ┌──────────┐
                  │ 继续扫描  │ → 回到"寻找蓝图"
                  │ 下一个    │
                  └──────────┘
```

### 4.2 种植任务状态机

```
        ┌──────────┐
        │ 扫描温室  │
        └────┬─────┘
             │
    ┌────────┼────────┐
    ▼        ▼        ▼
  成熟?   缺水?    空地?
    │        │        │
    ▼        ▼        ▼
 收获    浇水     播种
    │        │        │
    ▼        ▼        ▼
 物资存    加水     取种子
 入仓库    动画     从仓库
    │        │        │
    └────────┼────────┘
             ▼
        ┌──────────┐
        │ 继续扫描  │ → 循环
        └──────────┘
```

### 4.3 搬运任务状态机

```
        ┌──────────────┐
        │ 扫描地面物资  │
        │ 或仓库溢出    │
        └──────┬───────┘
               │ 找到待搬运物资
               ▼
        ┌──────────────┐
        │ 走到物资位置  │
        └──────┬───────┘
               │ 到达
               ▼
        ┌──────────────┐
        │ 拾取（按负重  │
        │ 上限判断数量） │
        └──────┬───────┘
               │
               ▼
        ┌──────────────┐
        │ 走到目标仓库  │
        │（按物资类型）  │
        └──────┬───────┘
               │ 到达
               ▼
        ┌──────────────┐
        │ 存入 / 堆叠   │
        └──────┬───────┘
               │
               ▼
        ┌──────────────┐
        │ 还有物资？    │
        └──┬───────┬──┘
       有  │       │ 没有
           ▼       ▼
        继续搬运   结束任务
```

## 五、第一层：即时反应——中断系统

所有任务执行过程中，持续检查以下条件。一旦触发，**立即中断当前任务**。

### 5.1 中断条件与动作

| 触发条件 | 动作 | 优先级 |
|----------|------|--------|
| 体温 < 个人危险阈值 | 走到最近热源（锅炉房/火盆旁） → 等待体温恢复 | 最高 |
| 饥饿 ≥ 90% | 走到仓库 → 取食物 → 进食 | 最高 |
| 疲劳 ≥ 95% | 走到居住舱 → 睡觉 | 最高 |
| 重伤/濒死 | 走到医务室或呼叫医生 | 最高 |
| 基地被袭击 | 有武器的→战斗位；无武器的→躲藏 | 高 |
| 火灾 | 取水灭火 | 高 |
| 体温 < 警告阈值 | 缩短当前任务时间（如露天建造→改为室内搬运） | 中 |
| 饥饿 ≥ 70% | 当前任务完成后去进食 | 低 |
| 疲劳 ≥ 80% | 当前任务完成后去睡觉 | 低 |

### 5.2 热源寻找逻辑

```
当前体温 < 阈值
    │
    ├─ 基地内？
    │   ├─ 我在供暖房间里吗？
    │   │   ├─ 是 → 等待恢复（房间在供暖就不用动）
    │   │   └─ 否 → 寻路到最近的供暖房间
    │   │
    │   └─ 没有供暖房间？（全基地熄火）
    │       └─ 走到锅炉房，尝试启动锅炉（如果燃料够）
    │           或走到火盆旁手动生火
    │
    └─ 基地外？
        └─ 走向撤离点 / 寻找临时庇护所
```

## 六、状态系统与行为修饰

### 6.1 五条状态槽

```
健康：████████░░ 78%

士气：████████░░ 74%
疲劳：████░░░░░░ 41%
```

### 6.2 状态对行为的修饰

| 状态 | 阈值 | 行为修饰 |
|------|------|----------|
| 营养 < 30% | 饥饿 | 效率 ×0.7，优先找食物 |
| 营养 < 10% | 极度饥饿 | 效率 ×0.4，可能偷吃仓库物资 |
| 体温 < 40% | 寒冷 | 效率 ×0.5，频繁中断找热源 |
| 体温 < 20% | 严重冻伤 | 效率 ×0.2，移速 -50% |
| 疲劳 > 70% | 困倦 | 效率 ×0.8 |
| 疲劳 > 90% | 极度困倦 | 效率 ×0.3，可能在任何地方睡着 |
| 士气 < 30% | 低落 | 效率 ×0.7，可能怠工 |
| 士气 < 10% | 崩溃 | 随机行为：哭泣/破坏/攻击他人/叛逃 |

## 七、性格特质对 AI 的修饰

性格不只是文字标签，它直接修改 AI 的行为参数。

| 特质 | 行为修饰 |
|------|----------|
| **勤劳** | 疲劳阈值 +20%（更晚去睡觉），闲置概率 -50% |
| **懒惰** | 疲劳阈值 -20%，停工休息频率 +100% |
| **乐观** | 士气自然下降 -50%，低温士气惩罚减半 |
| **悲观** | 士气自然下降 +50%，低温士气惩罚加倍 |
| **自私** | 饥饿时优先拿最好食物；10%概率私藏搜刮物资 |
| **利他** | 优先治疗他人；与伤员同处一室时士气不降 |
| **勇敢** | 战斗时后撤阈值降低 |
| **胆小** | 战斗时更容易逃跑；敌人警报触发更快躲藏 |
| **酗酒** | 在仓库取物资时优先拿伏特加，喝醉后效率 ×0.6 |
| **失眠** | 睡觉恢复疲劳效率 -50%，夜间随机起床 |

## 八、关系系统对 AI 的修饰

幸存者之间的关系影响协作行为和事件触发。

| 关系 | 效果 |
|------|------|
| 好友 | 同处一个房间 → 士气缓慢上升；协作建造速度 +20% |
| 恋人 | 同处居住舱 → 士气大幅上升；一方受伤→另一方情绪波动 |
| 竞争对手 | 同处一个房间 → 士气下降；协作效率 -20% |
| 仇敌 | 可能引发争吵事件；战斗时互不支援 |

## 九、手动指令覆盖

玩家可以随时**右键点击幸存者 → 指派特定任务**：

```
手动指令优先级：
  强制移动 → 最高，立即中断一切
  强制工作 → 高，完成当前子步骤后切换
  加入探索队 → 高，走向集合点
  强制睡觉/进食 → 中，当前任务完成后执行
```

**手动 vs 自动的平衡**：
- 手动指派后，幸存者完成该任务 → **回到优先级表自动选择**（不会傻等）
- 如果玩家频繁手动指派同一个幸存者 → 该幸存者获得"被微管理"隐形标签 → AI 透明度提高（在状态栏显示"正在等待新指令…"）

## 十一、性能考虑

- **战略评估**：每游戏小时（约 1 分钟实时）触发一次，全幸存者串行评估即可
- **任务执行**：状态机驱动，不需要每帧完整评估
- **即时反应**：用事件驱动 + 每隔 1 秒轮询关键阈值
- **寻路**：使用预计算的导航网格，只在基地结构变化时重建

---

> 幸存者 AI 的核心哲学：**可预测但不愚蠢**。玩家应该能理解为什么某个幸存者做了某事，但不必事无巨细地下指令。环世界的魔法在于——小人的行为有时让你惊喜，有时让你抓狂，但从不让你觉得"这是个 bug"。
