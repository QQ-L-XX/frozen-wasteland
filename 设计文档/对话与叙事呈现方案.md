# 对话与叙事呈现方案

> 填补 P1 缺失项 | 2026-05-18 | v0.1

---

## 一、三类叙事载体

这个游戏的信息传递分三种载体，各有不同的 UI 形式和设计意图：

| 载体 | 触发方式 | 呈现形式 | 玩家操作 |
|------|---------|---------|---------|
| **幸存者对话** | AI 自主 / 事件触发 | 头顶气泡 + 底部对话条 | 被动阅读 |
| **叙事文本** | 搜刮中发现 / 故事种子 | 全屏阅读面板 | 主动翻阅 |
| **道德抉择** | 关键事件节点 | 全屏弹窗 + 选项 | 必须选择 |

---

## 二、幸存者对话

### 2.1 头顶气泡（日常对话）

```
用途：幸存者表达即时状态——不打断游戏，看一眼就懂

出现时机：
  · 开始执行任务时（"我去修管道"）
  · 状态变化时（"好饿..."）
  · 完成任务时（"墙壁建好了"）
  · 环境反应（"太冷了！"）
  · 社交互动（对其他幸存者的话）

视觉设计：
  ┌──────────────┐
  │ 我去修管道... │ ← 淡入，3-5秒后淡出，不阻塞操作
  └──────────────┘
       │
      幸存者头顶

  · 白色背景 + 深灰文字
  · 圆角矩形，最大宽度 200px
  · 不阻塞点击（点击穿过气泡）
  · 不同情绪可改变气泡颜色（红色=危险，蓝色=悲伤，金色=高兴）
```

### 2.2 底部对话条（重要对话 / 多人对话）

```
用途：当多个幸存者对话，或内容较长需要玩家注意时

触发：
  · 幸存者之间发生重要对话（冲突、合作、关系变化）
  · 玩家点击正在对话的幸存者 → 展开底部对话条查看完整记录
  · 事件触发的预设对话

视觉设计：
┌─────────────────────────────────────────────────────┐
│ 老兵：我们必须去南边找燃料。再不去就来不及了。          │
│ 医生：但北边的医院可能有药品——                       │
│ ──────────────────────────────────────────────────  │
│ [按空格继续] [自动播放 ▸]  [跳过 ▸▸]                  │
└─────────────────────────────────────────────────────┘

  · 固定底部，半透明深色背景
  · 说话者名字 + 头像（左侧小图标）
  · 文字逐字打印（typewriter 效果）
  · 不暂停游戏——对话进行时游戏仍在运行
  · 可选：语音（如果有预算）或无语音（纯文本）
```

### 2.3 对话生成方式：预设 + 标签替换

```
对话模板（设计师预设）：
  "[speaker.name]：[emotion]。[task_description]"

标签池：
  speaker.name → 幸存者名字
  emotion → 从情绪标签池随机抽（高兴/疲惫/焦虑/坚定…）
  task_description → 从当前任务类型映射（建造/种植/狩猎…）

示例模板：
  · "我去{task}了。需要{required_material}。"
  · "这天气...{weather_comment}"
  · "你看到{social_target.name}了吗？"

实际输出：
  · "我去修管道了。需要金属片。"
  · "这天气...越来越冷了。"
  · "你看到了吗？他今天一直站在外面。"

不是环世界式的完全随机拼接，而是预设 30-50 个模板 + 动态填充，
既保留了人物性格的一致性，又有足够的变化。
```

## 三、叙事文本

### 3.1 在搜刮中发现

```
触发：搜刮到日记、便条、照片、录音带时

呈现：全屏阅读面板

┌─────────────────────────────────────┐
│                                     │
│      ┌─────────────────────┐        │
│      │                     │        │
│      │    [日记内容]        │        │
│      │                     │        │
│      │  "第 47 天。          │        │
│      │   燃料快用完了。       │        │
│      │   汤姆说他在西北方向    │        │
│      │   看到了一座医院。      │        │
│      │   也许那里还有药品...   │        │
│      │   但他说那边已经有人    │        │
│      │   去过了。             │        │
│      │                      │        │
│      │   明天就走。"          │        │
│      │                     │        │
│      └─────────────────────┘        │
│                                     │
│  ┌──────────────────────────────┐   │
│  │ 📋 新线索：圣玛丽医院          │   │
│  │ → 已标记在世界地图上           │   │
│  │                              │   │
│  │ [关闭]  [加入收藏]            │   │
│  └──────────────────────────────┘   │
└─────────────────────────────────────┘

设计要点：
  · 深色背景 + 纸张纹理的中央面板（像在读一张真实的纸）
  · 关键信息（地点名、人名）自动高亮
  · 如果触发了故事种子 → 读完后自动标记地图或更新日志
  · "加入收藏" → 存入基地的"日记架"（可建造的设施，用于回顾已发现的叙事文本）
```

### 3.2 叙事日志（在基地中回顾）

```
基地中建造"书架"或"日志台"后：

玩家点击 → 打开叙事日志面板：

┌─────────────────────────────────────┐
│ 📚 发现的记录                        │
│                                     │
│ ▸ 便条 · Day 2 — 西北方向的医院      │
│ ▸ 日记残页 · Day 5 — 第 47 天       │
│ ▸ 照片 · Day 8 — 全家福（背面有字）   │
│ ▸ 录音带 · Day 12 — 实验室日志 #3    │
│                                     │
│ [阅读选中] [关闭]                    │
└─────────────────────────────────────┘

已读的叙事文本可以随时回顾。
未读的以 🔴 标记。
所有文本关联到发现它的时间和地点。
```

## 四、道德抉择

### 4.1 弹窗设计

```
触发：关键事件（陌生人敲门、发现竞争对手营地、是否救援等）

呈现：全屏弹窗，暂停游戏

┌─────────────────────────────────────────┐
│                                         │
│         一名陌生人出现在基地门口。         │
│                                         │
│  暴风雪中，一个裹着破布的身影靠在门框上。   │
│  他看起来很虚弱，嘴唇发紫，               │
│  但背上有一把步枪。                       │
│  他说他的营地在南边，已经两周没消息了。     │
│                                         │
│  ┌─────────────────────────────────┐   │
│  │ ○ 让他进来                       │   │
│  │   新幸存者加入。                   │   │
│  │   风险：可能是掠夺者间谍（概率 10%）  │   │
│  ├─────────────────────────────────┤   │
│  │ ○ 给他一些食物，让他走             │   │
│  │   消耗食物 5 单位，士气 +5         │   │
│  │   可能后续在其他地方再遇到他        │   │
│  ├─────────────────────────────────┤   │
│  │ ○ 赶走他                         │   │
│  │   安全。士气 -5。                 │   │
│  │   可能后续被报复（概率 20%）        │   │
│  └─────────────────────────────────┘   │
│                                         │
│  你的选择会影响接下来的故事走向。          │
│                                         │
└─────────────────────────────────────────┘

设计要点：
  · 用文字营造氛围（不说"一个NPC出现在门口"，而是描写画面）
  · 选项不是"好/坏"——每个选项有清晰的代价和概率
  · 概率信息可选择性显示（硬核模式可隐藏概率，只给文字描述）
  · 选择后不能反悔
  · 选择结果不是立即揭晓——几天后才知道（间谍暴露 / 他带着礼物回来）
```

### 4.2 抉择的后续反馈

```
选择"A：让他进来"

  Day N+2：新幸存者"杰克"加入了基地，帮忙搬运物资。
           → 他是一个好帮手。（80%概率的正常结果）

  或者：

  Day N+5：夜里仓库被盗，杰克消失了，同时还少了 20 单位的食物和一把猎枪。
           → 他是一个掠夺者间谍。（20%概率的坏结果）

  或者：

  Day N+10：杰克在搜刮时发现了稀有蓝图并带回来。
           → 他是一个隐藏的技师。（惊喜结果）

设计意图：让玩家记住的不是"我选了 A"，而是"那个暴风雪夜，我让他进来了，
         然后..."。故事是在选择之后展开的，不是在选择的瞬间结束的。
```

## 五、整体叙事节奏

```
游戏中的叙事密度随时间变化：

  第 1-2 天   几乎没有叙事——生存是第一主题。
             只有对话气泡和必要的系统提示。

  第 3-10 天  开始出现叙事种子。
             搜刮中发现便条和日记 → 标记新区域。
             幸存者之间产生初步关系（友好/摩擦）。

  第 10-20 天 叙事密度达到高峰。
             无线电信号 → 外部世界开始介入。
             道德抉择接连出现。
             幸存者个人故事线展开。

  第 20 天+   叙事由故事种子+事件链主导。
             不再有新种子持续涌入，
             但之前播下的种子开始开花结果。
```

---

> 叙事设计核心：**让玩家记住的不是系统弹出的文本，而是他们在恶劣环境中做过的选择**。最好的叙事是玩家自己的故事——"你还记得那次暴风雪夜，你让那个陌生人进来后发生了什么吗？"
