> 填补 GDD 缺失项 | 2026-05-18 | v0.1

## 一、三张地图的关系

```
【世界地图】宏观抽象 — 俯瞰冰封城市（5×5 km²）
   │
   ├─ 区域为节点，路径为连线
   └─ 每个节点对应一个可进入的局部场景
          │
          ▼
【局部场景】微观具体 — 50×50 ~ 100×100 格
   │
   └─ 室内模板+随机化 / 开放废墟程序化生成
          │
          ▼
【基地场景】核心永久 — 初始 50×50，可扩展到 200×200
   │
   └─ 地形有天然特征：岩石、冻土、冰裂、低洼
```

## 二、世界地图生成

### 2.1 结构：节点-连线图

```
        [军事区]
           │
    [商业街]──[工业区]
       │ ╲      │
       │  ╲     │
    [郊区B]─[基地]──[医院]
       │     │
       │   [郊区A]
       │
    [地铁入口]──[地铁深处]──[研究中心]
```

### 2.2 生成算法

```
算法：带约束的随机节点图

输入：SEED（世界种子）、MAP_SIZE=5×5 km²

步骤：
  1. 放置基地节点于地图中心偏下
  2. 生成 3 层同心环：
     · 环1（0.5-1.5 km）：郊区×3、商业街×2、小型仓库
     · 环2（1.5-3.0 km）：医院、工业区×2、学校、地铁入口
     · 环3（3.0-5.0 km）：军事区、研究中心、地铁深处
  3. 泊松盘采样撒节点（保证不重叠）
  4. Delaunay 三角剖分取边连通节点（保证无孤立）
  5. 赋距离值 + 10% 随机波动
  6. 分配区域类型模板（按环限制类型池）
```

### 2.3 区域节点属性

```
{
    id, type, position(相对基地偏移), 距离(km),
    燃料消耗, 风险等级, 枯竭度(0-100%),
    已知物资, 可能蓝图, 敌人类型,
    特殊标记(null | 求救信号 | 故事种子 | 商队路线),
    已发现(bool), 已解锁(bool)
}
```

### 2.4 视野迷雾

- 初始：基地 + 半径 0.3km
- 新区域发现：探索队成功返回 → 解锁目标 + 沿途区域
- 无线电扫描 → 随机标记远处区域（不自动解锁路径）
- 故事种子 → 直接标记特定区域
- 晴空窗口 → 视野半径翻倍

## 三、局部搜刮场景生成

### 3.1 两种策略

| 策略 | 适用 | 方法 |
|------|------|------|
| 模板+随机化 | 室内建筑（医院、公寓、军事基地） | 预设 3-5 个布局，随机选择 + 微调 |
| 程序化生成 | 开放废墟（商业街、工业区、荒野） | 规则 + 噪声驱动 |

### 3.2 室内模板系统

```
以"医院"为例（50×50 格）：

  ┌─────────────────────┐
  │     停车场（室外）    │
  ├──────┬──────┬───────┤
  │ 急诊  │ 走廊  │ 药房   │
  │ 大厅  │      │       │
  ├──────┤      ├───────┤
  │ 病房A │      │ 病房B  │
  ├──────┴──────┴───────┤
  │      地下室入口       │
  │   ┌───┐             │
  │   │隔离│  ← 故事种子  │
  │   │病房│   可能位置   │
  │   └───┘             │
  └─────────────────────┘

随机化参数：
  · 走廊是否畅通（70%畅通 / 30%部分堵塞）
  · 病房门是否上锁（40%概率）
  · 物资点位置在房间内随机偏移 ±2 格
  · 敌人初始位置在预设"敌人区"内随机放置
  · 故事种子从候选点中选 1-2 个激活
  · 室外 + 破窗房间有积雪（减速区）
```

### 3.3 开放废墟：程序化生成

```
算法：简化的 Wave Function Collapse (WFC)

地块类型：
  · 建筑废墟（不可进入，可攀爬/翻越）
  · 半开放商店（可进入，有屋顶但墙体部分塌）
  · 街道（开阔，积雪覆盖）
  · 车辆残骸（可搜刮，提供掩护）

生成步骤：
  1. 地图尺寸（如 60×40 格）
  2. WFC 生成地块布局（~0.5 秒）
  3. 在"可进入"地块内撒物资点（密度 × 区域类型）
  4. 放置敌人巡逻路线（沿街道网格）
  5. 用 Perlin Noise 决定积雪深度和冰面
  6. 放置撤离点（地图边缘 2-4 个候选，选最近且未堵塞的）
```

### 3.4 场景种子示例

```
区域：圣玛丽医院
模板：hospital_medium（3 变体中的 1 个）
随机种子：第 5 次探索，枯竭度 23%

生成结果：
  · 选模板 B（药房在二楼而非一楼）
  · 1楼病房A 门锁了（需开锁器）
  · 地下室隔离病房激活故事种子"被焊死的门"
  · 物资点 12 个（满额 18，枯竭度削掉 6）
  · 敌人：6 个冻饿者（走廊3+病房B2+地下室1）
  · 暴风雪窗口：14 分钟
```

## 四、基地场景生成

### 4.1 初始地块（50×50 格）

```
Perlin Noise 生成地形：

     等高线示意：
     ┌──────────────────┐
     │    ··            │  ← 北侧裸露岩石（不可挖掘）
     │  ····            │
     │ ······    ▓▓     │  ← 东侧小冰裂（可架桥）
     │  ·····    ▓▓     │
     │   ···      ▓▓    │
     │    ·       ▓▓    │
     │                 │
     │     🏠          │  ← 初始木棚位置
     │                 │
     │          ░░░    │  ← 南侧低洼（天然防风，热损 -15%）
     │         ░░░     │
     └──────────────────┘

地形类型：
  · 平整雪地：直接建造，成本最低
  · 冻土：需挖掘（×2 工时），但地下保温 +10%
  · 岩石：不可挖掘，天然屏障（防御 +20%）
  · 冰裂：无法建造，需架桥（金属×3）
  · 低洼地：天然防风，热损 -15%
```

### 4.2 可扩展边界

```
扩展代价：
  · 平整雪地：1 工时/格
  · 挖掘冻土：4 工时/格
  · 爆破岩石：需炸药（搜刮获得）
  · 架桥过冰裂：金属×3 + 木材×2 / 格

最大基地尺寸：200×200 格（性能上限）
```

### 4.3 推荐建屋点

开局系统标记 2-3 个推荐位置（综合地势、岩石距离、冰裂距离评估），玩家可自选。

## 五、寻路与导航网格

### 基地
- NavMesh 构建：可通行格 / 墙壁岩石冰裂=阻断 / 门=条件
- 基地结构变化时增量更新 NavMesh
- A* + Jump Point Search (JPS) 优化网格寻路

### 搜刮场景
- 同 NavMesh 方案
- 额外动态障碍：敌人身体、掉落物、可破坏障碍
- 攀爬点：废墟堆、翻窗（需工具或属性检定）

## 六、推荐数据结构

```typescript
// 世界地图
interface WorldMap {
  nodes: RegionNode[];
  edges: { from: string; to: string; distance: number }[];
}

// 局部场景（运行时）
interface LocalScene {
  width: number; height: number;
  tiles: Tile[][];
  lootPoints: LootPoint[];
  enemies: Enemy[];
  evacuationPoints: Point[];
  storySeed: StorySeed | null;
  weatherWindow: number;
}

// 基地
interface BaseGrid {
  width: number; height: number;
  cells: BaseCell[][];
  rooms: Room[];
  pipes: Pipe[];
}

interface BaseCell {
  terrain: 'snow' | 'permafrost' | 'rock' | 'ice_crack';
  building: BuildingPiece | null;
  temperature: number;
  insulationRate: number;
}
```

## 七、性能预算

| 地图 | 规模 | 内存 | 生成时间 |
|------|------|------|---------|
| 世界地图 | ~20节点 | <1 KB | <10 ms |
| 局部(室内) | 50×50=2500格 | ~200 KB | <100 ms |
| 局部(开放) | 80×50=4000格 | ~300 KB | <300 ms |
| 基地 | 初始50×50→最大200×200 | 1-4 MB | <500 ms |

Cocos Creator 里完全没压力。

---

> 核心原则：**用模板保证可玩性，用随机保证重玩性**。纯随机 = 可能生成死局；纯手绘 = 重复游玩无新鲜感。模板约束 + 参数随机化是独立游戏的标准做法。
