# 事件系统 · 触发与编排

> 关联 GDD 第十章 | 2026-05-18 | v0.1

---

## 一、设计哲学

事件不是随机生成的"惊喜"，而是**对玩家状态的回应**。好的事件系统让玩家感觉"这个世界在看着我"——不是机械地掷骰，而是对当前处境做出有逻辑的反馈。

### 核心原则
- **状态驱动**：事件概率和内容与玩家当前处境挂钩
- **后果可见**：事件发生后，你过几天能看到它的连锁影响
- **不频繁轰炸**：每天最多 1-2 个事件，让玩家有时间应对

---

## 二、事件分类

| 类别 | 频率 | 驱动方式 | 举例 |
|------|------|---------|------|
| 天气事件 | 自然节奏 | Perlin Noise | 暴风雪、短暂晴空、极寒夜 |
| 基地事件 | 1 次/1-3 天 | 概率+状态修正 | 管道破裂、火灾、冲突 |
| 外部事件 | 1 次/3-7 天 | 概率+进度修正 | 掠夺者袭击、求救信号、商队 |
| 幸存者事件 | 人物驱动 | 性格+关系触发 | 崩溃、觉醒、失踪、求婚 |
| 道德抉择 | 关键节点 | 手动触发（故事种子） | 收留陌生人、救援还是掠夺 |

## 三、事件调度器

### 3.1 整体调度流程

```
每游戏小时：
  │
  ├─ 天气系统更新（Perlin Noise → 温度/风速/降雪）
  │   └─ 满足暴风雪条件？→ 触发天气事件
  │
  ├─ 幸存者状态检查（全员遍历）
  │   ├─ 士气 < 阈值？→ 可能触发崩溃事件
  │   ├─ 健康 < 阈值？→ 触发疾病进展
  │   ├─ 关系事件检查（有好/恶关系积累）
  │   └─ 特殊技能觉醒检查
  │
  └─ 每日零点：
      ├─ 掷骰：是否有基地事件？
      ├─ 掷骰：是否有外部事件？
      └─ 推进所有已触发的故事种子
```

### 3.2 事件冷却机制

```
事件触发后 → 进入冷却期 → 冷却期内同类事件概率 ×0.1

冷却期：
  正面事件（晴空、商队）：2-4 天
  负面事件（袭击、火灾）：1-2 天
  幸存者事件（崩溃、觉醒）：3-5 天
```

防止"运气差就连续挨打"或"运气好就每天好事不断"。

## 四、天气事件编排

### 4.1 Perlin Noise 天气曲线

```
温度曲线（归一化）：
  基础 = PerlinNoise( day × 0.1, seed )  [周期约 6-10 天]
  叠加 = PerlinNoise( day × 0.4, seed2 ) [周期约 1-2 天，短波动]
  趋势 = 0.02 × day  [缓慢下降，越往后越冷]
  
  最终室外温度 = 基础×20 + 叠加×5 - 趋势 - 15
  → 日常范围：-10°C ~ -35°C，暴风雪可到 -50°C
```

### 4.2 暴风雪触发条件

```
if (风速 > 20 AND 降雪量 > 0.6):
    暴风雪概率 = (风速-20) × 0.05 + (降雪量-0.6) × 0.8
    if (roll < 暴风雪概率):
        触发暴风雪事件
```

### 4.3 特殊窗口——"晴空"

暴风雪之间的短暂天晴是重要的探索窗口：

```
晴空触发：上一次暴风雪结束后 2-4 天内随机出现一次
晴空期间：室外温度 +10°C，可见度 100%，搜刮窗口翻倍
持续时间：0.5-1 天

设计意图：让玩家有"必须抓住这个窗口"的紧迫感
```

## 五、基地事件

### 5.1 管道破裂

```
触发概率：
  基础 5%/天
  + 室外温度 < -20°C → +10%
  + 管道总长度 > 30 格 → +5%
  - 维修优先级表上有 ≥ 2 人 → -3%

触发后：
  ┌──────────────────────────────┐
  │ ⚠ 管道破裂！                  │
  │ 3 号管道（锅炉房→温室）泄漏     │
  │ 温室正在失去供暖...            │
  │                              │
  │ 需要：金属×2 + 破布×1 修复     │
  │ 修复耗时：30 秒（技师自动响应）  │
  │                              │
  │ [手动指派] [自动处理]          │
  └──────────────────────────────┘

后果：
  未及时修复 → 对应房间温度持续下降
  24 小时内未修 → 管道冻裂（需要更换，材料 ×3）
```

### 5.2 火灾

```
触发概率：
  基础 2%/天
  + 使用火盆 → +5%
  + 使用废塑料做燃料 → +8%
  - 有烟囱 → -1%

触发后：
  指定房间起火 → 所有人自动中断任务去灭火
  需要水源（雪/水）→ 每份水减少火势 20%
  火势 100% → 房间烧毁，相邻房间可能被波及
```

### 5.3 幸存者冲突

```
触发条件（不靠概率，靠条件累积）：
  - 两个"竞争对手"或"仇敌"关系的人共处同一房间 > 2 小时
  - 基地整体士气 < 40%
  - 食物紧张（存量 < 3 天）
  
  满足 2/3 条件 → 触发冲突事件

后果：
  两个当事人士气 -20
  可能需要玩家介入调停（抉择：偏袒一方？各打五十大板？）
```

## 六、外部事件

### 6.1 掠夺者袭击

```
触发概率：
  基础 3%/天
  + 积累物资 > 100 格 → +8%
  + 无线电持续开机 → +5%（信号暴露位置）
  - 有围墙 → -2%
  - 有自动炮塔 → -5%

袭击流程：
  1. 预警（如果有哨兵/感知高的幸存者）：提前 30 秒通知
  2. 掠夺者从地图边缘出现 → 向基地移动
  3. 有武器的幸存者自动就位战斗
  4. 掠夺者目标：仓库和锅炉房
  5. 击退或被杀 → 结束后搜刮掠夺者尸体（武器+弹药）
```

### 6.2 求救信号

```
触发：无线电持续开机时，3-5 天一次概率

┌──────────────────────────────┐
│ 📡 无线电收到信号...           │
│ "这里是北区避难所，我们有 4 人   │
│  食物耗尽，医疗用品短缺。       │
│  如果有人听到...请回应..."     │
│                              │
│  ○ 回应并派出救援队（消耗燃料）  │
│     → 获得新幸存者，+士气       │
│  ○ 回应但要求用物资交换         │
│     → 可能获得蓝图/信息         │
│  ○ 忽略                       │
│     → 士气 -5（仅医生知道）     │
└──────────────────────────────┘
```

### 6.3 商队

```
触发：第 10 天以后，3-7 天概率

商队出现 → 临时交易 UI：
  玩家可以出售多余物资，购买稀有物品
  商队接受：子弹 > 药品 > 伏特加 > 电池

商队类型：
  · 拾荒商：出售基础物资，收食品/药品
  · 武器商：出售武器弹药，只收子弹（以物易物）
  · 科技商：出售蓝图/电子元件，收稀有零件
```

## 七、幸存者事件

### 7.1 精神崩溃

```
触发条件：
  士气 < 20% 持续 2 天以上

崩溃表现（随机）：
  · 暴怒：攻击最近的人/破坏家具
  · 崩溃：原地哭泣，无法行动，持续半天
  · 逃避：躲进房间不出去，拒绝工作
  · 自毁：跑到室外雪地（有冻死风险）

恢复：
  需要其他幸存者安抚（社交互动）+ 改善条件
  持续 1-2 天后自行恢复，士气重置为 30%
```

### 7.2 技能觉醒

```
触发条件：
  某幸存者在同一工作上持续了 20+ 天
  AND 相应属性 ≥ 7

觉醒 → 获得一个永久被动技能：
  · 建造大师：建造速度 +30%
  · 绿拇指：温室产出 +50%
  · 战地医生：医疗效果 +40%
  · 猎手：狩猎发现稀有物资概率 ×2
```

### 7.3 失踪

```
触发条件（极低概率，用于叙事驱动）：
  某幸存者在探索时未返回（如果 B 层探索由 AI 执行）
  或在基地附近独自行动时触发

失踪 → 在世界地图上出现新标记"最后已知位置"
  → 玩家可选择派出搜救队
  → 可能找回（受伤但存活）或找到尸体（+日记）
```

## 八、道德抉择事件

这些是**故事种子触发的关键节点**，每个都有持久后果：

### 8.1 陌生人

```
┌─────────────────────────────────┐
│ 一名陌生人在暴风雪中出现在基地门口。│
│ 他看起来很虚弱，但背上有一把步枪。  │
│ 他说他的营地在南边，已消失 2 周。   │
│                                 │
│  ○ 让他进来 → 新幸存者加入         │
│    风险：可能是掠夺者间谍（小概率）  │
│  ○ 给他一些物资让他走 → +士气      │
│    消耗食物 5 单位                 │
│  ○ 拒绝并赶走 → 安全               │
│    -士气，可能后续被报复            │
└─────────────────────────────────┘
```

### 8.2 竞争对手营地

```
发现另一个幸存者营地（通过无线电或探索）。
他们有燃料但缺药品。

  ○ 提议交易 → 建立贸易路线，双方受益
  ○ 偷袭抢夺 → 获得大量燃料，但士气崩溃，可能被报复
  ○ 不干预 → 不做任何事
```

## 九、事件链——连锁反应

好的事件系统让一个事件引发另一个，形成故事弧：

```
例：一条可能的事件链

Day 5: 无线电收到求救信号 → 你派出救援队
Day 6: 救回一名中年女性（曾是护士）→ 新幸存者加入
Day 8: 护士说她的丈夫还在南边的医院 → 世界地图出现新标记
Day 12: 搜刮医院时发现她丈夫已变成冻饿者 → 护士崩溃事件
Day 14: 护士从崩溃中恢复 → 觉醒"战地医生"技能
Day 20: 护士组织建立医务室升级 → 基地医疗能力翻倍

——这是 emergent storytelling 的核心机制
```

## 十、事件编辑器（开发工具思路）

为了方便测试和调试，可以考虑一个简单的**事件触发器**：

```
开发模式控制台：
  > event.trigger("blizzard")
  > event.trigger("raid", intensity=3)
  > event.force_story_seed("hospital_basement")
  > survivor.morale("老兵", -30)
```

这对调优事件节奏非常有用。
