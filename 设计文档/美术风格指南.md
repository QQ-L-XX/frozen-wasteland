# 美术风格指南

> 方向性参考 | 2026-05-18 | v0.1

---

## 一、总体视觉方向

### 风格选择：低多边形像素风
- 不是纯像素，也不是纯手绘
- **清晰可辨的图形 + 有限色板 + 粗线条 + 轻微纹理噪点**
- 类似《Kingdom: Two Crowns》/《Darkest Dungeon》的色调处理，但更偏冷

### 为什么
- 俯视角管理游戏需要可读性，像素太糊，手绘太贵
- 低多边形 + 有限色板 = 辨识度高 + 成本可控

## 二、色调板

### 室外（冷）
```
天空/雪地：#C8D8E8 → #E8F0F8 → #A0B0C0
废墟/建筑：#5A5A6E → #3A3A4E → #2A2A3A
阴影/裂痕：#1A1A2E（近黑蓝）
暴风雪叠加：白色 + 降低透明度
```

### 基地（暖）
```
火光/锅炉：  #FF8C42 → #FF6B35 → #D44A1A
木材/墙面：  #8B6F5E → #6B5040 → #4A3828
照明范围：   #FFD700（暖黄光晕）
```

### 对比规则
```
场景        背景色         前景/UI         强调色
室外        蓝白灰         深灰/白         暖橙(火光)
室内基地    深褐/暗        暖黄/白         冰蓝(冷区警示)
搜刮场景    灰+暗红褐      蓝白            琥珀(物资点)
暴风雪      全白+低α       几乎不可见      红(危险)
```

## 三、角色设计

- **3-4 头身小比例**，头部略大便于表情识别
- 身高约 48-64 px（1080p 俯视角下）
- 通过颜色+装备+状态特效区分幸存者
- 每个幸存者有一个随机分配的主色（围巾/帽子/外套）

### 状态视觉反馈
```
正常：       站立，轻微呼吸
寒冷：       发抖 + 呼出白气粒子
受伤：       跛行 + 红色闪烁
冻伤：       身体蓝紫色
士气低落：   低头，移速慢
高兴：       偶尔跳跃
```

## 四、UI 风格

- **极简终端风**：细线条、半透明面板、无衬线字体
- 类似《冰汽时代》顶部温度条的冷静感

### UI 色板
```
面板背景：  rgba(10,15,25,0.85)  深蓝黑半透明
文字主色：  #D8E0E8               淡蓝白
强调色：    #FF8C42（警告）或 #4FC3F7（信息）
进度条：    #FF6B35（暖橙→红）
```

### 字体
- 中文：思源黑体 Regular（现代、冷感）
- 数字/英文：等宽字体（JetBrains Mono）
- 叙事文本：思源宋体（日记/便条用衬线增加"旧纸"感）

## 五、粒子与环境特效

| 特效 | 用途 | 实现 |
|------|------|------|
| 雪花 | 室外持续飘落，暴风雪密集 | 粒子系统，白/浅蓝 |
| 呼出白气 | 幸存者在寒冷中 | 小团白色粒子从头部上升 |
| 火光闪烁 | 火盆/锅炉 | 暖橙光晕脉冲 |
| 冷气流线 | 热量泄漏可视化 | 蓝白半透明流线 |
| 霜冻蔓延 | 屏幕边缘暴风雪预警 | 从边缘向内生长的冰晶 |
| 烟气 | 烟囱排放 | 灰白烟柱上升 |

## 六、场景美术分层（搜刮场景）

```
Layer 1 — 地面（雪/瓦砾）
Layer 2 — 建筑结构（墙壁、门）
Layer 3 — 可交互物（物资点图标、敌人 —— 有颜色对比）
Layer 4 — 环境覆盖（积雪、冰面、血迹 —— 半透明叠加）
Layer 5 — 光照（火盆暖光 / 窗外冷光）
Layer 6 — UI（HUD、噪音条、倒计时）
```

## 八、原型阶段美术优先级

| 优先级 | 需要 | 最低方案 |
|--------|------|---------|
| P0 | 地块纹理、木墙、火盆 sprite | 纯色方块 + 色块 |
| P0 | 幸存者基础 sprite（1个通用） | 色块小人 |
| P0 | 物资点图标（箱/柜/尸体 3种） | 简单形状 |
| P1 | 雪花粒子、火光光晕 | 粒子系统 |
| P1 | UI 面板框架 | 矩形 + 半透明 |
| P2 | 完整动画、装饰物、环境细节 | — |

> 美术的核心任务：让玩家看一眼就知道"哪里冷、哪里暖"。冷暖对比不是美观问题——是 gameplay 信息。
