# 新手体验 · 早期一天流程

> 关联 GDD 全篇 | 2026-05-18 | v0.1

---

## 设计目标

用游戏内前 2 天（约真实时间 20-30 分钟）让玩家理解所有核心系统，不做冗长教程，而是通过**自然的生存压力驱动学习**。

---

## 第一天：醒来就是危机

### 游戏开始 · 镜头淡入

```
黑屏。风声。

画面渐亮——一个狭小的木棚内部。
三个人蜷缩在火盆旁。火苗微弱。
窗外是永无止境的白色。

┌─────────────────────────────────┐
│  🕐 第 1 天 · 06:00 · -28°C      │  ← 顶部信息条
│                                 │
│  [木棚内部俯视图]                 │
│  · 3 个幸存者围坐在火盆旁          │
│  · 火盆的木材快烧完了              │
│  · 角落堆着少量物资               │
│                                 │
│  ┌─────────────────────────┐   │
│  │ 老兵：我们得想办法。        │   │
│  │ 食物够 3 天。但燃料只够      │   │
│  │ 撑到今天下午。              │   │
│  └─────────────────────────┘   │
│                                 │
│  [点击任意位置继续]              │
└─────────────────────────────────┘
```

**设计意图**：不言自明地向玩家展示三个事实——(1)你很冷，(2)食物有限，(3)燃料更少。不需要弹教程窗。

---

### 阶段 A：加燃料——第一次交互

玩家点击火盆 → 弹出燃料选项：

```
┌──────────────────┐
│  🔥 火盆         │
│  剩余：1 小时     │
│                  │
│  [添加燃料]      │
│  · 木材 ×5 (库存 50)
│  [确认]          │
└──────────────────┘
```

点击确认 → 老兵自动走过去，把木材放进去，火光亮了一些。

```
💬 技师：火盆撑不了太久。我们需要煤。
💬 医生：或者一间更好的屋子。
```

**系统层发生了什么**：玩家完成了第一次"物资消耗"操作。燃料库存从 50→45。火盆计时器从 1 小时延长到 6 小时。

---

### 阶段 B：查看工作优先级——理解自动化

玩家此时可能开始探索界面。点击屏幕底部的人物头像。

```
点击老兵头像 → 弹出信息面板：

┌──────────────────────────────┐
│ 🪖 老兵 · 力量 7 · 感知 8     │
│ 健康 ████████░░ 85%          │
│ 营养 ██████████ 95%          │
│ 体温 ██████░░░░ 60%          │
│ 士气 ████████░░ 78%          │
│                               │
│ 当前任务：闲置                 │
│                               │
│ [工作优先级] ← 点击            │
└──────────────────────────────┘
```

点击工作优先级：

```
┌──────────────────────────────┐
│ 工作优先级——老兵              │
│                               │
│ 建造  3 ▓▓▓▓▓░░░░            │
│ 狩猎  4 ▓▓▓▓░░░░░            │
│ 种植  / ░░░░░░░░░            │
│ 医疗  2 ▓▓▓▓▓▓░░░            │
│ 维修  2 ▓▓▓▓▓▓░░░            │
│ 搜刮  1 ▓▓▓▓▓▓▓▓▓ ← 最优先   │
│ 搬运  3 ▓▓▓▓▓░░░░            │
│                               │
│ [关闭]                        │
└──────────────────────────────┘
```

玩家看一眼就理解了——这些小人会自己干活，我不需要一个个指挥。这是环世界式管理的核心认知，在这里无声建立。

**不弹教程窗。**玩家如果愿意，可以拖拽调整优先级。不调也没关系——老兵默认就会优先搜刮。

### 阶段 C：第一次建造——木墙

火盆加完燃料后，技师会自动开始寻找建造任务（因为他建造优先级=1）。

但如果没有任何蓝图，他会进入闲置。玩家可能在摸索界面时点击了屏幕底部的**建造按钮**：

```
点击 [🔨 建造] → 底部展开建造菜单：

┌──────────────────────────────────────┐
│ 墙壁 │ 地板 │ 门 │ 窗 │ 设施 │ 管道  │
│                                      │
│ 可用：                               │
│ [木墙] 木材×3 保温40%                 │
│ [火盆] 石材×3                        │
│                                      │
│ (更多建筑需要蓝图解锁)                 │
└──────────────────────────────────────┘
```

玩家选择木墙 → 鼠标变为放置模式 → 点击地面拖拽规划：

```
玩家拖拽画出一段墙 → 出现半透明预览（蓝色=可建造）

┌────────────┐
│  现有木棚   │
│  (15格)    │
└────┬───────┘
     ├──[新墙预览]──┤
     │   ▓▓▓▓▓▓▓▓  │ ← 蓝色半透明
     │   ▓▓▓▓▓▓▓▓  │
     └─────────────┘

左下角显示：
  需要木材：12（库存：45）
  预计保温率：40%
  [确认建造]
```

确认 → 技师接到建造任务。

```
💬 技师：我去造。大约需要 2 分钟（实时）。
```

技师走向目标格，开始敲击动画。进度条出现在建筑上方。

**系统层发生了什么**：
- 木材库存 45→33
- 建造任务加入技师的任务队列（因为他的建造优先级=1）
- 技师自动从"闲置"状态切换到"建造"状态机
- 墙壁建成后，木棚 → 木屋（保温率从 40%→60%，如果有绝缘材料）

### 阶段 D：食物告急——驱动第一次探索

约 2-3 分钟（游戏时间推进到中午），系统弹出第一个"软提醒"：

```
┌──────────────────────────────┐
│ 📊 每日报告——第 1 天中午      │
│                              │
│ 食物存量：27 单位（够 3 天）   │
│ 燃料存量：木材 39 + 煤 20     │
│   → 供暖限界：约 2 天         │
│                              │
│ ⚠ 医生提醒：药品不足，有人受   │
│   伤的话无法处理。             │
│                              │
│ 💡 提示：世界地图上标注了附近   │
│   的郊区住宅——那里可能还有物资。│
│                              │
│ [打开世界地图] [继续管理基地]   │
└──────────────────────────────┘
```

这不是强制教程，而是自然的情境压力。食物和燃料都指向同一个结论——该出门了。

### 阶段 G：第一次搜刮场景——学习搜刮

进入郊区住宅的局部场景：

```
┌──────────────────────────────────────┐
│  ⏱ 天气晴好 · 搜刮窗口：15 分钟        │
│  🔊 噪音：░░░░░░░░░░ 0%               │
│  🎒 背包：0 / 20 格                    │
│                                       │
│    [郊区住宅内部俯视图]                 │
│    厨房  客厅  卧室                     │
│    🗄️×2  📦×2  🗄️×2                  │
│                                       │
│  老兵 [████████░░]                    │
└──────────────────────────────────────┘
```

玩家点击厨房的🗄️ → 第一个搜刮交互弹窗：

```
┌──────────────────────────────┐
│  🗄️ 厨房柜子                  │
│                               │
│  安静搜刮 → 噪音+2 | 100%收益  │
│  快速搜刮 → 噪音+8 | 70%收益   │
│                               │
│  (暴力破拆不可用——没有撬棍装   │
│   备在工具槽)                  │
│                               │
│  [安静搜刮] [快速搜刮]         │
└──────────────────────────────┘
```

玩家选安静搜刮 → 8 秒倒计时 → 获得：

```
✅ 罐头×3
✅ 破布×2

背包：5 / 20 格
```

继续搜刮客厅、卧室 → 背包逐渐填满。

**关键教学时刻——噪音累积**：

搜刮 3-4 个物资点后，噪音条上升到 30% 左右。屏幕边缘微微泛红：

```
🔊 噪音：░░░░░▓▓▓▓▓ 34%
⚠ 老兵：我听到外面有动静...
```

一个冻饿者从地图边缘出现 → 老兵自动进入警戒状态。

**这是轻度战斗教学**——玩家点击冻饿者 → 老兵自动攻击（撬棍）→ 3 次打击后冻饿者倒地。

```
💬 老兵：只是其中一个。最好动作快点。
```

玩家理解了两件事：(1)搜刮产生噪音，(2)噪音引来敌人，(3)战斗是自动的但需要玩家点击目标。

### 阶段 H：撤退——第一次归来

背包 18/20 格，搜刮窗口还剩 8 分钟。玩家可以继续搜刮剩下的物资点，也可以提前撤退。如果背包满了，系统会弹出提示：

```
┌──────────────────────────────┐
│  🎒 背包已满（20/20）         │
│                               │
│  [返回基地] [丢弃部分物资]     │
└──────────────────────────────┘
```

返回 → 老兵走到撤离点 → 15 秒倒计时 → 安全撤离。

## 第二天：选择出现了

### 阶段 J：清晨报告——新危机

第二天早上，画面从基地夜晚过渡到白天。弹出每日报告：

```
┌──────────────────────────────────┐
│ 📊 第 2 天 · 06:00 · -32°C        │
│                                  │
│ ⚠ 木墙建造完成！保温率提升。       │
│    但现在只有火盆供暖——燃料消耗     │
│    比昨天更快了（房间大了）。        │
│                                  │
│ ⚠ 昨夜极寒（-38°C），燃料消耗超预期 │
│    煤存量：14 单位（够 1.5 天）    │
│                                  │
│ 💡 老兵：我在搜刮时发现这个...      │
│    [一张皱巴巴的便条]              │
│    "西北方向有座医院——可能有药品   │
│     和更好的医疗设备。但那边去的人  │
│     多，已经快被搬空了。"          │
│                                  │
│  → 世界地图出现新标记：圣玛丽医院   │
│    风险：中 · 枯竭度：23%          │
│    距离：1.2 km / 燃料 3          │
│                                  │
│ [打开世界地图] [继续]              │
└──────────────────────────────────┘
```

**设计意图**：
- 玩家刚学会搜刮 → 系统马上抛出一个"更有价值的远目标"（医院）
- 但距离更远（燃料更多）→ 引出燃料管理的深层考量
- "枯竭度 23%"暗示这里已被别人搜过 → 暗示世界上有其他幸存者
- 便条是第一个"故事种子"——叙事从这里开始发芽

### 阶段 K：策略抉择——玩家第一次面对多路径

此时玩家的选择开始分叉：

```
路径 A：再去郊区A搜刮（安全但收益递减——枯竭度 27%）
路径 B：派老兵去商业街（新区域，风险中，物资多）
路径 C：开始建造温室（需要玻璃和土壤，需要搜刮）
路径 D：什么都不做，等天气好转（晴空窗口）

→ 每一种选择都有后果
```

**新手引导的边界**：
- 前 2 天已经教了：加燃料、看 UI、建造、世界地图、搜刮、战斗、撤退
- 第 3 天起，系统不再弹出"提示"，玩家完全自主决策
- 但后果自然呈现——选 A 可能资源不够，选 B 可能遇到危险，选 C 需要耐心

---

## 新手体验设计的核心原则

| 原则 | 实现方式 |
|------|---------|
| **不弹教程窗** | 所有教学通过对话气泡、自然情境、UI 暗示完成 |
| **压力驱动学习** | 燃料快没了 → 你自然学会了加燃料。食物快没了 → 你自然学会了搜刮 |
| **给软提示不给硬任务** | "医生提醒药品不足" vs "任务：去 X 地点拿药"。前者尊重玩家自主 |
| **前 20 分钟建立循环** | 加燃料 → 建造 → 探索 → 搜刮 → 返回 → 看到基地变好——这个循环在第一天就完整跑通 |
| **第 2 天开启深度** | 故事种子、多区域选择、建设 vs 探索的权衡——在基础循环建立后立刻出现 |
